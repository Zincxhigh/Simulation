PSX 80 Minute Patcher

Patch PS1/PS2 Games for 700MB CD-R Compatibility

About

PSX80MP is a tool by Alex Free that patches PlayStation game images to work on early PS2 models when burned to 80 minute / 700MB CD-Rs.

It fixes a hardware bug found in early PS2 consoles that causes crashes or boot failures when using higher capacity discs.

Download

Version 2.0.1 Stable

  • psx80mp-v2.0.1-windows-i686-static.zip Portable Release For Windows 95 OSR 2.5 and above, Pentium CPU minimum (32 bit)

  • psx80mp-v2.0.1-windows-x86_64-static.zip Portable Release For x86_64 Windows (64 bit)

  • psx80mp-v2.0.1-linux-i386-static.zip Portable Release For Linux 3.2.0 and above, 386 CPU minimum (32 bit)

  • psx80mp-v2.0.1-linux-i386-static.deb Deb package file For Linux 3.2.0 and above, 386 CPU minimum (32 bit)

  • psx80mp-v2.0.1-linux-x86_64-static.zip Portable Release For x86_64 Linux 3.2.0 and above (64 bit)

  • psx80mp-v2.0.1-linux-x86_64-static.deb Deb package file for x86_64 Linux 3.2.0 and above (64 bit)

How It Works

Early PS2 consoles expect lower capacity discs. When using 700MB discs, the laser can overshoot data, causing read errors.

PSX80MP fixes this by adding dummy data (CDDA silence sectors), preventing the laser from reading invalid areas.

Usage

Drag & Drop Method

Drag your BIN file onto the executable.

Command Line

psx80mp <data_track.bin>

Note: Usually patch track 1. If it fails, try track 2.

Why This Matters

Previously, users had to rely on smaller CD-Rs (650MB). This tool allows modern 700MB discs to work reliably on affected systems.

  • ESP8266 host method

  • If it sounds too good → it’s a scam

  • 2022 – Kernel + WebKit exploit
  • 2022 – IPv6 exploit

Applying ROM Patches

How to Patch Games Across Different Systems

Overview

Many consoles with interchangeable games support community-made ROM patches and hacks.

These patches modify games using different tools depending on the patch format.

Most ROM hacks can be found on ROMhacking.net.

IPS Patches

Steps
  • Download Lunar IPS and launch it
  • Extract the .ips file if compressed
  • Click Apply IPS Patch
  • Select the .ips file
  • Select your game ROM
  • Success message should appear

IPS patches are simple and do not verify if the correct ROM is used.

They are limited to 16MB patch size.

BPS Patches

Steps
  • Download Beat v1 and launch it
  • Extract the .bps or .bpm file
  • Click Apply Patch
  • Select the patch file
  • Select your ROM
  • Choose output location and name

BPS patches include safety checks to ensure correct ROM matching.

PPF Patches

Steps
  • Ensure you have a single .bin or .iso file
  • Make a backup of your game file
  • Extract the .ppf file
  • Download and open PPF-O-MATIC
  • Select your game file
  • Select the patch file
  • Click Apply

Used mainly for PlayStation and PS2 games, but also works on other systems.

Multi-track games must be merged using tools like Binmerge.

xdelta Patches

Steps
  • Extract the .xdelta file
  • Download and open Delta Patcher
  • Select your original ROM
  • Select the patch file
  • Click Apply patch

xdelta patches are widely used across many systems and support advanced patching.

Troubleshooting

  • Ensure you are using the correct ROM version
  • Check for multiple game revisions
  • Verify checksum or hash if provided
  • Wrong ROM = patch will fail

Swiss FTP

Transfer Files from PC to Console Using FTP

Overview

Swiss includes a built-in FTP client that allows you to download files directly from your PC to your console.

Unlike most systems, Swiss connects to your PC instead of acting as a server.

Setup FTP Server

Steps
  • Create a folder on your PC to store files
  • Download and run xlight.exe
  • Allow firewall access on private network if prompted
  • Click New Virtual Server → OK
  • Open the server → go to Public Path
  • Add:
    • Virtual Path: /
    • Real Path: Your folder location
    • Permissions: Enable all

Create User

Steps
  • Click User List → Add
  • Set username & password (e.g. gamecube)
  • Set home directory to /
  • Enable all permissions
  • Click OK to save

Start Server

Steps
  • Select your server
  • Click Run Server
  • Right-click → Server Network Interfaces
  • Note your PC IP (e.g. 192.168.0.5)

Connect in Swiss

Steps
  • Open Swiss network settings
  • Set FTP Host IP to your PC IP
  • Enter username and password
  • Press B then A to change device
  • Select File Transfer Protocol
  • Connection should show your PC files

Important Notes

  • PC and console must be on same network
  • Firewall must allow FTP server
  • Correct IP is required for connection
  • Swiss acts as client, not server

Installing Custom Firmware (CFW)

Extend Your Nintendo Switch Functionality

Overview

This guide covers setting up Custom Firmware (CFW) on the Nintendo Switch to enable homebrew and advanced features.

CFW is not permanently installed and must be launched each time depending on your console type.

Section I - Check Exploit Compatibility

Details
  • Check model number on the back of the console
  • Patched models (require modchip):
    • HAC-001(-01) (Switch v2)
    • HDH-001 (Switch Lite)
    • HEG-001 (Switch OLED)
  • Unpatched model: HAC-001 (2017–2019)
  • Serial number can help determine exploitability
  • Modchipped consoles show "No SD card" screen without SD card

Section II - Requirements

What You Need
  • Latest version of Atmosphere (CFW)
  • fusee.bin payload file
  • Latest version of Hekate
  • microSD card (minimum 8GB, recommended 64GB+)
  • USB SD card reader

Atmosphere enables homebrew, while Hekate provides tools like NAND backup and boot management.

Section III - File Setup

Steps
  • Insert microSD card into PC using reader
  • Extract files to SD card root:
    • atmosphere
    • bootloader
    • switch
    • hekate_ctcaer_x.x.x.bin
  • Place fusee.bin into bootloader/payloads
  • Modchipped: Rename hekate file to payload.bin
  • Unpatched: Delete hekate file after use
  • Safely eject SD card and insert into Switch

Section IV - Finalizing Setup

Steps
  • Boot into Hekate (auto for modchip, manual for unpatched)
  • Go to Tools → Backup eMMC
  • Backup:
    • BOOT0 & BOOT1
    • RAW GPP (full NAND)
  • Transfer backup to a safe location
  • Delete NAND backup from SD card to free space

Section V - Optional Setup

Recommended
  • Format SD card to FAT32 (reduces corruption risk)
  • Go to Tools → Partition SD Card
  • Create emuMMC if desired

Final Step

Go to Home → Payloads → select fusee.bin to launch Atmosphere.

You are now running Custom Firmware on your Switch.

Important Notes

  • CFW is not permanently installed
  • Must be reloaded after shutdown (especially on unpatched consoles)
  • Always keep NAND backups safe to prevent bricking

XBHDM

Softmod Your Xbox & Enable Homebrew

Overview

XBHDM is a process used to softmod your Xbox console, allowing you to run homebrew software and game backups.

This method works by unlocking the Xbox hard drive, connecting it to a PC, modifying its files directly, and then re-locking it using the console’s HDD key or user password.

Important Notes

  • Requires unlocking and re-locking the Xbox hard drive
  • Uses HDD key or EEPROM backup for security
  • Can be used on new drives if correct key/password is available
  • Avoid hotswapping (unsafe and outdated method)

Section I - Requirements

What You Need
  • Windows PC
  • FATXplorer 3.0 Beta
  • XBHDM Build.zip
  • Way to access Xbox HDD key (EEPROM backup or HDD password)
  • Tools to open/disassemble the Xbox

Section II - Preparation

Steps
  • Download and extract FATXplorer and XBHDM to separate folders
  • Disassemble the Xbox to access the hard drive
  • Connect the hard drive to your PC

Section III - Unlock & Backup

Steps
  • Open FATXplorer and go to HDD Security Tools
  • Select the Xbox hard drive
  • Unlock it using the HDD password or EEPROM backup
  • Mount the C partition and back it up
  • Also back up the E partition (contains saves & DLC)

Section IV - Install Softmod

Steps
  • Drag and drop XBHDM softmod files onto the hard drive
  • Replace any existing files if prompted

Section V - Re-lock & Reinstall

Steps
  • Use FATXplorer to re-lock the hard drive
  • Safely unmount the drive from your PC

Bad Update

Xbox 360 Hypervisor Exploit (17559)

Overview

Bad Update is a non-persistent, software-only hypervisor exploit for all Xbox 360 models running dashboard version 17559.

It allows running unsigned code when combined with tools like XeUnshackle or FreeMyXe.

The exploit is temporary and will reset after the console is powered off.

Supported Entry Points

  • Exploited Xbox Avatar (ABadAvatar)
  • Rock Band Blitz (trial or full)
  • Tony Hawk's American Wasteland (NTSC/PAL/RF)

This exploit has around an 80% success rate and may require multiple attempts.

Section I - Required Items

Requirements
  • Xbox 360 (dashboard 17559)
  • Supported game OR ABadAvatar
  • Xbox360BadUpdate (if using a game)
  • Extra avatar data (required for ABadAvatar if avatars appear gray)
  • USB drive (minimum 64MB, FAT32 formatted)
  • 512MB+ USB (required for Rock Band Blitz)
  • XeUnshackle (recommended) or FreeMyXe
  • XeXMenu and additional homebrew (Aurora, FreeStyle 3, etc.)

Steps

  • If using ABadAvatar, no payload replacement is required
  • Add any extra homebrew to the USB drive

Section III - Disable Auto Sign-In (ABadAvatar)

  • Turn off Auto Sign-In

  • Newer versions remove need to patch XEX files
